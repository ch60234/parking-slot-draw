# 櫻花金馬之櫻 - 社區機車位抽籤系統

這是一套專為「櫻花金馬之櫻」社區設計的純前端機車位抽籤系統。系統無需伺服器或資料庫，只需透過瀏覽器開啟網頁即可執行，確保抽籤過程的公開、透明與絕對隨機性。

## ✨ 核心特色功能

* **雙模式切換 (防呆機制)**
  * **🧪 測試模式**：背景呈現淡綠色，可無限次點擊抽籤，方便事前測試或向住戶展示流程。此模式下不會出現匯出按鈕。
  * **⚠️ 正式模式**：背景切換為淡粉紅色，點擊抽籤後按鈕將自動鎖定以防誤觸。若需重新抽籤必須經過雙重確認解鎖。
* **嚴謹的驗證機制 (防竄改)**
  * 抽籤完成後系統會立即進行自我稽核，確保「車位無重複分配」且「無任何戶別遺漏」。
  * 驗證通過後，會透過 **SHA-256 演算法** 即時產生一組獨一無二的「數位抽籤指紋」，作為該次抽籤結果的數位憑證。
* **名單匯出與建檔**
  * 在「正式模式」下抽籤成功後，可點擊「📥 匯出名單」下載 CSV 檔案（支援 Excel 開啟，無亂碼）。
  * 匯出檔案的最下方會自動附上「數位抽籤指紋」與「匯出時間」，供日後公告與查核使用。
* **清晰的視覺排版**
  * 戶別名單採用 **A棟、B棟、C棟 獨立直行並排顯示**，無論是在電腦螢幕或投影幕上，住戶都能極快找到自己的戶別與結果。
* **歷屆紀錄查詢**
  * 專屬的歷史紀錄頁面 (`History.html`)，採用手風琴 (Accordion) 收合設計，越新的紀錄會自動置頂並展開。

## 🚀 使用說明

1. **啟動系統**：雙擊 `Index.html` 即可在任何現代瀏覽器 (Chrome, Edge, Safari) 中開啟本系統。
2. **進行抽籤**：
   * 進入網頁後，預設為「測試模式」。
   * 正式抽籤時，點選「⚠️ 正式模式」，然後按下「🎲 開始第二屆抽籤」。
3. **匯出結果**：抽籤完畢並看到「系統內部驗證通過」的提示後，點擊「📥 匯出名單」儲存 CSV 檔案。
4. **查詢歷史**：透過畫面上方的選單，可隨時切換至「📜 歷史紀錄」頁面查詢歷屆資料。

## 🛠️ 維護與資料更新

若未來有新增屆期或是社區戶別變動，請使用文字編輯器打開 `javascript/data.js` 檔案：

1. **修改基礎資料**：更新 `firstTermData` 陣列內的物件 `{ "unit": "戶別", "slot": "車位號碼" }`。
2. **新增歷史紀錄**：
   * 將最新一次抽籤結果的完整陣列，複製並格式化。
   * 打開 `History.html`，找到 `historicalRecords` 陣列，並在**最上方**插入新的一筆 `{ id: "...", title: "...", period: "...", data: [...] }` 紀錄即可。

本系統所有的排序（依棟別、樓層、戶號）均會由程式自動處理。
